The Gradio web interface provides a comprehensive, user-friendly way to generate music with ACE-Step. This guide covers all features and workflows.

## Getting Started

### Launching the Interface

<CodeGroup>
```bash Basic Launch
python app.py
```

```bash With Pre-initialization
python app.py --config acestep-v15-turbo --init-llm
```

```bash Custom Port
python app.py --port 7860
```
</CodeGroup>

The Gradio interface automatically opens in your browser at `http://localhost:7860`.

### Interface Layout

The interface is organized into several key sections:

<Steps>
  <Step title="Settings (Collapsed Accordion)">
    Service configuration, model selection, DiT/LM parameters, and output options
  </Step>
  
  <Step title="Generation Tab">
    Main workspace with generation mode selector and input fields
  </Step>
  
  <Step title="Results Section">
    Generated audio playback, scoring, and batch navigation
  </Step>
  
  <Step title="Training Tab">
    Dataset builder and LoRA training (optional)
  </Step>
</Steps>

## Service Configuration

### Model Selection

Before generating music, configure your models:

| Setting | Description |
|---------|-------------|
| **Main Model Path** | DiT model configuration (e.g., `acestep-v15-turbo`, `acestep-v15-turbo-shift3`) |
| **Device** | Processing device: `auto` (recommended), `cuda`, or `cpu` |
| **Checkpoint File** | Select a trained model checkpoint if available |

### 5Hz Language Model Configuration

<Note>
**Adaptive Defaults**: All LM settings are automatically configured based on your GPU's VRAM tier. The system pre-selects optimal models and backends for your hardware.
</Note>

| Setting | Description |
|---------|-------------|
| **5Hz LM Model Path** | Select the language model. Available models are filtered by GPU tier (6-8GB GPUs show only 0.6B; 24GB+ GPUs show all sizes) |
| **5Hz LM Backend** | `vllm` (faster, recommended for NVIDIA ≥8GB VRAM), `pt` (PyTorch, universal), or `mlx` (Apple Silicon). On GPUs &lt;8GB, backend is restricted to `pt`/`mlx` |
| **Initialize 5Hz LM** | Load the LM during initialization (required for thinking mode). Automatically disabled on GPUs ≤6GB |

### Performance Options

<Warning>
**Tier-Aware Settings**: Offload, quantization, and compile options are automatically set based on your GPU tier. Manual overrides may cause memory issues.
</Warning>

| Setting | Description |
|---------|-------------|
| **Use Flash Attention** | Enable for faster inference (requires flash_attn package) |
| **Offload to CPU** | Offload models to CPU when idle to save GPU memory. Auto-enabled on GPUs &lt;20GB |
| **Offload DiT to CPU** | Specifically offload the DiT model to CPU. Auto-enabled on GPUs &lt;12GB |
| **INT8 Quantization** | Reduce VRAM footprint with INT8 weight quantization. Auto-enabled on GPUs &lt;20GB |
| **Compile Model** | Enable `torch.compile` for optimized inference. Enabled by default on all tiers |

### LoRA Adapter

<Warning>
LoRA adapters cannot be loaded on quantized models due to a compatibility issue between PEFT and TorchAO. Set **INT8 Quantization** to **None** before loading LoRA.
</Warning>

| Setting | Description |
|---------|-------------|
| **LoRA Path** | Path to trained LoRA adapter directory |
| **Load LoRA** | Load the specified LoRA adapter |
| **Unload** | Remove the currently loaded LoRA |
| **Use LoRA** | Enable/disable the loaded LoRA for inference |

### Initialization

Click **Initialize Service** to load models. The status box displays:
- Detected GPU tier and VRAM
- Maximum allowed duration and batch size
- Warnings about incompatible settings that were auto-corrected

After initialization, **Audio Duration** and **Batch Size** sliders update to reflect tier limits.

## Generation Modes

The **Generation Mode** radio selector determines your workflow. Turbo and SFT models offer four modes; base models add three more.

### Simple Mode

Quick, natural language-based music generation.

<Steps>
  <Step title="Select Simple Mode">
    Choose **Simple** in the Generation Mode radio
  </Step>
  
  <Step title="Enter Description">
    Write a natural language description in the "Song Description" field
  </Step>
  
  <Step title="Configure Options">
    - Check "Instrumental" if you don't want vocals
    - Select preferred vocal language (optional)
  </Step>
  
  <Step title="Create Sample">
    Click **Create Sample** to generate caption, lyrics, and metadata
  </Step>
  
  <Step title="Review & Generate">
    Review the generated content, then click **Generate Music**
  </Step>
</Steps>

**Example descriptions:**
- "a soft Bengali love song for a quiet evening"
- "upbeat electronic dance music with heavy bass drops"
- "melancholic indie folk with acoustic guitar"
- "jazz trio playing in a smoky bar"

<Tip>
Click the 🎲 button to load a random example description.
</Tip>

### Custom Mode

Full control over all generation parameters (text2music).

<Tabs>
  <Tab title="Basic Workflow">
    1. Select **Custom** in the Generation Mode radio
    2. Manually fill in Caption and Lyrics fields
    3. Optionally upload Reference Audio for style guidance
    4. Set optional metadata (BPM, Key, Duration, etc.)
    5. Click **Generate Music**
  </Tab>
  
  <Tab title="With LM Enhancement">
    1. Fill in basic Caption and Lyrics
    2. Click **Format** to enhance your input using the LM
    3. Review enhanced content
    4. Configure advanced settings as needed
    5. Click **Generate Music**
  </Tab>
</Tabs>

### Remix Mode

Transform existing audio while maintaining melodic structure but changing style.

<CodeGroup>
```text Workflow
1. Select Remix in Generation Mode
2. Upload Source Audio (the song to remix)
3. Write Caption describing target style
4. Optionally modify Lyrics
5. Adjust Remix Strength (0.0-1.0)
   - Higher = closer to original structure
6. Click Generate Music
```

```text Use Cases
- Creating cover versions
- Style transfer
- Generating variants of a song
```
</CodeGroup>

### Repaint Mode

Regenerate a specific time segment while keeping the rest intact.

<Steps>
  <Step title="Select Repaint Mode">
    Choose **Repaint** in the Generation Mode radio
  </Step>
  
  <Step title="Upload Source">
    Upload the audio file to edit
  </Step>
  
  <Step title="Set Time Range">
    - **Repainting Start**: Start time in seconds
    - **Repainting End**: End time in seconds (use -1 for end of file)
  </Step>
  
  <Step title="Describe Changes">
    Write a Caption describing the desired content for the repainted section
  </Step>
  
  <Step title="Generate">
    Click **Generate Music**
  </Step>
</Steps>

**Use cases:** Fixing problematic sections, changing lyrics in a segment, extending songs.

### Extract Mode (Base Model Only)

Extract/isolate a specific instrument track from mixed audio.

<Note>
This mode requires a base model (e.g., `acestep-v15-base`). It will be auto-downloaded if missing.
</Note>

1. Select **Extract** in the Generation Mode radio
2. Upload Source Audio
3. Select the **Track Name** to extract from the dropdown
4. Click **Generate Music**

**Available tracks:** vocals, backing_vocals, drums, bass, guitar, keyboard, percussion, strings, synth, fx, brass, woodwinds

### Lego Mode (Base Model Only)

Add a new instrument track to existing audio.

1. Select **Lego** in the Generation Mode radio
2. Upload Source Audio
3. Select the **Track Name** to add
4. Write a Caption describing track characteristics
5. Click **Generate Music**

### Complete Mode (Base Model Only)

Complete partial tracks with specified instruments (auto-arrangement).

1. Select **Complete** in the Generation Mode radio
2. Upload Source Audio
3. Select multiple **Track Names** to add
4. Write a Caption describing the desired style
5. Click **Generate Music**

## Input Parameters

### Audio Inputs

| Field | Description |
|-------|-------------|
| **Reference Audio** | Optional audio for style/timbre guidance (visible in Custom mode) |
| **Source Audio** | Required for Remix, Repaint, Extract, Lego, Complete modes |
| **Convert to Codes** | Extract 5Hz semantic codes from source audio |

#### LM Codes Hints (Advanced)

Pre-computed audio semantic codes can be pasted here to guide generation. Use the **Transcribe** button to analyze codes and extract metadata. This is an advanced feature for controlling melodic structure without uploading source audio.

### Music Caption

The text description of the desired music. Be specific about:
- Genre and style
- Instruments
- Mood and atmosphere
- Tempo feel (if not specifying BPM)

**Example:** "upbeat pop rock with electric guitars, driving drums, and catchy synth hooks"

<Tip>
Click 🎲 to load a random example caption.
</Tip>

### Lyrics

Enter lyrics with structure tags:

```
[Verse 1]
Walking down the street today
Thinking of the words you used to say

[Chorus]
I'm moving on, I'm staying strong
This is where I belong

[Verse 2]
...
```

**Instrumental checkbox:** Generate instrumental music regardless of lyrics content.

**Vocal Language:** Select language for vocals. Use "unknown" for auto-detection.

**Format button:** Enhance caption and lyrics using the 5Hz LM.

### Optional Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| **BPM** | Auto | Tempo in beats per minute (30-300) |
| **Key Scale** | Auto | Musical key (e.g., "C Major", "Am", "F# minor") |
| **Time Signature** | Auto | Time signature: 2 (2/4), 3 (3/4), 4 (4/4), 6 (6/8) |
| **Audio Duration** | Auto/-1 | Target length in seconds (10-600). -1 for automatic |
| **Batch Size** | 2 | Number of audio variations to generate (1-8) |

## Advanced Settings

### DiT Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| **Inference Steps** | 8 | Denoising steps. Turbo: 1-20, Base: 1-200 |
| **Guidance Scale** | 7.0 | CFG strength (base model only). Higher = follows prompt more |
| **Seed** | -1 | Random seed. Use comma-separated values for batches |
| **Random Seed** | ✓ | When checked, generates random seeds |
| **Audio Format** | mp3 | Output format: mp3, flac |
| **Shift** | 3.0 | Timestep shift factor (1.0-5.0). Recommended 3.0 for turbo |
| **Inference Method** | ode | ode (Euler, faster) or sde (stochastic) |
| **Custom Timesteps** | - | Override timesteps (e.g., "0.97,0.76,0.615,0.5,0.395,0.28,0.18,0.085,0") |

### LM Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| **LM Temperature** | 0.85 | Sampling temperature (0.0-2.0). Higher = more creative |
| **LM CFG Scale** | 2.0 | LM guidance strength (1.0-3.0) |
| **LM Top-K** | 0 | Top-K sampling. 0 disables |
| **LM Top-P** | 0.9 | Nucleus sampling (0.0-1.0) |
| **LM Negative Prompt** | "NO USER INPUT" | Negative prompt for CFG |

### Generation Options

| Option | Default | Description |
|--------|---------|-------------|
| **Think** | Enabled | Enable 5Hz LM for code generation and metadata |
| **ParallelThinking** | - | Enable parallel LM batch processing |
| **CaptionRewrite** | - | Let LM enhance the input caption |
| **AutoGen** | - | Automatically start next batch after completion |

## Results Section

### Generated Audio

Up to 8 audio samples are displayed based on batch size. Each sample includes:

- **Audio Player** - Play, pause, and download
- **Send To Src** - Send audio to Source Audio input for further processing
- **Save** - Save audio and metadata to JSON file
- **Score** - Calculate perplexity-based quality score
- **LRC** - Generate lyrics timestamps (LRC format)

### Details Accordion

Expand "Score & LRC & LM Codes" to view:
- **LM Codes** - The 5Hz semantic codes for this sample
- **Quality Score** - Perplexity-based quality metric
- **Lyrics Timestamps** - LRC format timing data

### Batch Navigation

| Control | Description |
|---------|-------------|
| **◀ Previous** | View the previous batch |
| **Batch Indicator** | Shows current batch position (e.g., "Batch 1 / 3") |
| **Next Batch Status** | Shows background generation progress |
| **Next ▶** | View next batch (triggers generation if AutoGen is on) |

### Restore Parameters

Click **Apply These Settings to UI** to restore all generation parameters from the current batch. Useful for iterating on good results.

## Tips and Best Practices

### For Best Quality

<Accordion title="Quality Optimization">
  1. **Use thinking mode** - Keep "Think" checkbox enabled for LM-enhanced generation
  2. **Be specific in captions** - Include genre, instruments, mood, and style details
  3. **Let LM detect metadata** - Leave BPM/Key/Duration empty for auto-detection
  4. **Use batch generation** - Generate 2-4 variations and pick the best
</Accordion>

### For Faster Generation

<Accordion title="Speed Optimization">
  1. **Use turbo model** - Select `acestep-v15-turbo` or `acestep-v15-turbo-shift3`
  2. **Keep inference steps at 8** - Default is optimal for turbo
  3. **Reduce batch size** - Lower batch size for quick results
  4. **Disable AutoGen** - Manual control over batch generation
</Accordion>

### For Consistent Results

<Accordion title="Consistency">
  1. **Set a specific seed** - Uncheck "Random Seed" and enter a seed value
  2. **Save good results** - Use "Save" to export parameters for reproduction
  3. **Use "Apply These Settings"** - Restore parameters from a good batch
</Accordion>

## Troubleshooting

<AccordionGroup>
  <Accordion title="No audio generated">
    - Check that the model is initialized (green status message)
    - Ensure 5Hz LM is initialized if using thinking mode
    - Check the status output for error messages
  </Accordion>
  
  <Accordion title="Poor quality results">
    - Increase inference steps (for base model)
    - Adjust guidance scale
    - Try different seeds
    - Make caption more specific
  </Accordion>
  
  <Accordion title="Out of memory">
    The system includes automatic VRAM management (VRAM guard, adaptive VAE decode, auto batch reduction). If OOM still occurs:
    - Reduce batch size manually
    - Enable CPU offloading (should be auto-enabled for GPUs &lt;20GB)
    - Enable INT8 quantization (should be auto-enabled for GPUs &lt;20GB)
    - Reduce LM batch chunk size
    - See GPU_COMPATIBILITY.md for recommended settings per tier
  </Accordion>
  
  <Accordion title="LM not working">
    - Ensure "Initialize 5Hz LM" was checked during initialization (disabled by default on GPUs ≤6GB)
    - Check that a valid LM model path is selected (only tier-compatible models are shown)
    - Verify vllm or PyTorch backend is available (vllm restricted on GPUs &lt;8GB)
    - If the LM checkbox is grayed out, your GPU tier does not support LM — use DiT-only mode
  </Accordion>
</AccordionGroup>
