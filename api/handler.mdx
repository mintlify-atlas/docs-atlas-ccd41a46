`AceStepHandler` is the main handler class that encapsulates all data processing and business logic as a bridge between the model and UI. It manages model initialization, generation, and various audio processing operations.

## Key Methods

### initialize_service

Initialize the ACE-Step service with model components.

```python
handler.initialize_service(
    project_root: str,
    config_path: str,
    device: str = "cuda",
    offload_to_cpu: bool = False,
    quantization: Optional[str] = None,
    use_mlx_dit: bool = False,
    use_mlx_vae: bool = False,
)
```

**Parameters:**

<ParamField path="project_root" type="str" required>
  Path to project root directory.
</ParamField>

<ParamField path="config_path" type="str" required>
  Path or name of model config (e.g., "acestep-v15-turbo").
</ParamField>

<ParamField path="device" type="str" default="cuda">
  Device for inference ("cuda", "cpu", "mps", "xpu", "rocm").
</ParamField>

<ParamField path="offload_to_cpu" type="bool" default="False">
  Enable CPU offloading to save VRAM.
</ParamField>

<ParamField path="quantization" type="Optional[str]" default="None">
  Quantization mode: "int8_weight_only", "fp8_weight_only", "w8a8_dynamic", or None.
</ParamField>

<ParamField path="use_mlx_dit" type="bool" default="False">
  Enable MLX acceleration for DiT (macOS Apple Silicon only).
</ParamField>

<ParamField path="use_mlx_vae" type="bool" default="False">
  Enable MLX acceleration for VAE (macOS Apple Silicon only).
</ParamField>

**Example:**

```python
from acestep.handler import AceStepHandler

handler = AceStepHandler()
handler.initialize_service(
    project_root="/path/to/project",
    config_path="acestep-v15-turbo",
    device="cuda",
    offload_to_cpu=False,
    quantization="int8_weight_only",
)
```

## Usage Example

```python
from acestep.handler import AceStepHandler
from acestep.llm_inference import LLMHandler
from acestep.inference import GenerationParams, GenerationConfig, generate_music

# Initialize handler
dit_handler = AceStepHandler()
llm_handler = LLMHandler()

# Initialize services
dit_handler.initialize_service(
    project_root="/path/to/project",
    config_path="acestep-v15-turbo",
    device="cuda",
    offload_to_cpu=False,
    quantization="int8_weight_only",
)

llm_handler.initialize(
    checkpoint_dir="/path/to/checkpoints",
    lm_model_path="acestep-5Hz-lm-0.6B",
    backend="vllm",
    device="cuda"
)

# Use with high-level API
params = GenerationParams(
    caption="upbeat electronic dance music",
    bpm=128,
    duration=30,
)

config = GenerationConfig(
    batch_size=2,
    audio_format="flac",
)

result = generate_music(
    dit_handler,
    llm_handler,
    params,
    config,
    save_dir="/output"
)

if result.success:
    for audio in result.audios:
        print(f"Generated: {audio['path']}")
```

## Mixin Functionality

`AceStepHandler` inherits functionality from multiple mixins, organized by responsibility:

### Core Generation
- `GenerateMusicMixin`: Main generation orchestration
- `DiffusionMixin`: Diffusion model operations
- `ServiceGenerateMixin`: Service-level generation

### Conditioning
- `ConditioningTextMixin`: Text condition processing
- `ConditioningEmbedMixin`: Embedding generation
- `ConditioningMaskMixin`: Mask generation
- `ConditioningBatchMixin`: Batch conditioning
- `ConditioningTargetMixin`: Target condition handling

### Audio Processing
- `IoAudioMixin`: Audio I/O operations
- `VaeEncodeMixin`, `VaeDecodeMixin`: VAE encoding/decoding
- `VaeEncodeChunksMixin`, `VaeDecodeChunksMixin`: Chunked VAE operations
- `AudioCodesMixin`: Audio code processing

### Model Management
- `InitServiceMixin`: Service initialization
- `LoraManagerMixin`: LoRA adapter management
- `MemoryUtilsMixin`: Memory optimization utilities

### MLX Acceleration (macOS)
- `MlxDitInitMixin`: MLX DiT initialization
- `MlxVaeInitMixin`: MLX VAE initialization
- `MlxVaeEncodeNativeMixin`, `MlxVaeDecodeNativeMixin`: MLX VAE operations

### Utilities
- `BatchPrepMixin`: Batch preparation
- `PaddingMixin`: Padding utilities
- `ProgressMixin`: Progress tracking
- `MetadataMixin`: Metadata handling
- `TaskUtilsMixin`: Task-specific utilities
- `PromptMixin`: Prompt processing
- `LyricScoreMixin`, `LyricTimestampMixin`: Lyrics processing
- `TrainingPresetMixin`: Training preset handling

## Environment Variables

- `ACESTEP_DISABLE_TQDM`: Set to "1", "true", or "yes" to disable tqdm progress bars
- `ACESTEP_DEBUG_STATS`: Set to "1", "true", or "yes" to enable debug statistics
- `TOKENIZERS_PARALLELISM`: Automatically set to "false" to avoid fork warnings
