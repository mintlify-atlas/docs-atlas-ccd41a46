ACE-Step 1.5 is an open-source music generation model that runs locally on your GPU. This guide will get you from zero to your first generated song in just a few steps.

## Prerequisites

Before you begin, ensure you have:

- **Python 3.11 or 3.12** installed
- **CUDA-capable GPU** (recommended) - also supports MPS, ROCm, Intel XPU, or CPU
- **4GB+ VRAM** for basic usage (more VRAM enables larger language models)

<Note>
  ROCm on Windows requires Python 3.12 specifically, as AMD officially provides Python 3.12 wheels only.
</Note>

## Installation

<Steps>
  <Step title="Install uv package manager">
    First, install the `uv` package manager:

    <CodeGroup>
      ```bash macOS / Linux
      curl -LsSf https://astral.sh/uv/install.sh | sh
      ```

      ```powershell Windows
      powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
      ```
    </CodeGroup>
  </Step>

  <Step title="Clone and install ACE-Step">
    Clone the repository and install dependencies:

    ```bash
    git clone https://github.com/ACE-Step/ACE-Step-1.5.git
    cd ACE-Step-1.5
    uv sync
    ```

    This will install all required dependencies automatically.
  </Step>

  <Step title="Launch the Gradio UI">
    Start the web interface:

    ```bash
    uv run acestep
    ```

    Models will auto-download on first run. Open your browser to:
    
    **http://localhost:7860**

    <Note>
      The UI automatically selects the best model configuration for your GPU. All settings (LM model, backend, offloading, quantization) are pre-configured based on your available VRAM.
    </Note>
  </Step>
</Steps>

## Generate Your First Song

Once the Gradio UI is running:

1. **Enter a simple prompt** in the text input field, such as:
   - "upbeat electronic dance music with strong bass"
   - "acoustic guitar ballad with emotional vocals"
   - "jazz piano solo with smooth rhythm"

2. **Click Generate** and wait a few seconds

3. **Listen to your creation** - the audio player will appear with your generated music

That's it! You've just generated professional-quality music locally on your machine.

## What Model Should I Use?

The UI automatically selects the best configuration, but here's what runs based on your GPU:

| Your GPU VRAM | LM Model Used | Notes |
|---------------|---------------|-------|
| **≤6GB** | None (DiT only) | LM disabled; uses INT8 quantization + CPU offload |
| **6-8GB** | `acestep-5Hz-lm-0.6B` | Lightweight LM with PyTorch backend |
| **8-16GB** | `acestep-5Hz-lm-0.6B` or `1.7B` | 0.6B for 8-12GB, 1.7B for 12-16GB |
| **16-24GB** | `acestep-5Hz-lm-1.7B` | 4B available on 20GB+ |
| **≥24GB** | `acestep-5Hz-lm-4B` | Best quality, all models fit without offload |

## Alternative: Launch the REST API

If you prefer programmatic access, launch the REST API server instead:

```bash
uv run acestep-api
```

The API will be available at **http://localhost:8001**
